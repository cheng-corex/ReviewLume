# 用户指南

## 1. 创建审核包

1. 打开 Git 项目。
2. 运行 `ReviewLume: Create Review Pack`。
3. 选择审核范围：staged、unstaged 或 commit range。
4. 选择审核模式。
5. 检查自动纳入的文件。
6. 添加必要的需求文档或实施报告。
7. 查看敏感扫描结果。
8. 预览最终审核内容。
9. 导出 Markdown 或复制提示。

## 2. 审核模式选择

- 快速：小修复、单文件改动。
- 标准：普通功能、多个相关文件。
- 高风险：并发、数据库、备份恢复、生命周期、权限和兼容性。
- 自定义：完全手动选择。

## 3. 使用浏览器 AI

第一版推荐：

1. 点击“打开 AI 网页”。
2. 粘贴审核提示。
3. 上传或粘贴 Review Pack。
4. 手动确认当前模型和审核档位。
5. 手动发送。

ReviewLume 不保证第三方网页模型的可用性、准确性或数据处理政策。

## 4. 导入审核回答

1. 复制 AI 的完整回答。
2. 运行 `ReviewLume: Import Review Response`。
3. 粘贴回答。
4. 检查自动识别的结论和问题。
5. 调整严重程度和状态。
6. 保存审核报告。

## 5. 修复闭环

对每个问题设置：

- Open：尚未处理。
- Fixed：已修复。
- Rejected：确认是误报。
- Needs review：需要二次确认。

全部处理后生成复核请求，只发送：

- 首次问题清单。
- 修复摘要。
- 新 diff。
- 必要关联文件。

## 6. 隐私建议

- 不要发送生产数据库。
- 不要发送真实用户数据。
- 不要强制加入被 BLOCK 的文件。
- 审核前仔细检查预览。
- 内部项目应确认组织的数据使用规则。
- 使用第三方 AI 网站前阅读其隐私政策。

## 7. 常见问题

### 为什么不直接读取整个项目？

完整项目通常包含大量无关内容和敏感数据。ReviewLume鼓励最小必要上下文。

### ReviewLume 会调用 AI API 吗？

核心版不会。用户自行选择浏览器 AI 或其他方式处理审核包。

### ReviewLume 会自动修改代码吗？

第一版不会。它只生成审核上下文并管理结果。

### 审核结果一定正确吗？

不一定。AI 输出必须作为辅助意见，最终仍需开发者验证。

# P9 浏览器桥接人工验收清单

## 验收前提

- 使用 PR #21 对应分支构建并安装最新 VSIX。
- 以开发者模式加载仓库内 `browser-extension/`。
- 不导入真实账号凭据，不在测试记录中粘贴 Cookie、Session、Token 或网页回答正文。
- 至少准备一个已存在的 ReviewLume `reviewId` 和一段不含敏感信息的测试提示。

## 1. 基础启动与监听边界

1. 执行 `ReviewLume: Start Browser Bridge`。
2. 确认提示的地址为 `http://127.0.0.1:<随机端口>`。
3. 重启扩展宿主后再次启动，确认端口可以变化。
4. 确认服务未监听 `0.0.0.0`、局域网地址或公网地址。

通过标准：桥接仅在用户显式执行命令后启动，且只监听 loopback 随机端口。

## 2. 一次性配对

1. 执行 `ReviewLume: Pair Browser Extension`。
2. 在浏览器扩展弹窗输入桥接地址、配对码并选择一个目标站点。
3. 完成一次成功配对。
4. 使用相同配对码再次尝试配对。
5. 生成新配对码，等待其过期后再尝试使用。

通过标准：首次配对成功；重复使用和过期配对码均被拒绝；错误信息不泄露会话令牌或提示正文。

## 3. Optional Host Permission

分别对 ChatGPT、Claude、Gemini 执行：

1. 在未授权站点权限时尝试填入。
2. 由用户主动选择站点并批准权限。
3. 检查浏览器扩展权限页面。
4. 撤销该站点权限并再次尝试。

通过标准：默认不持有 AI 站点权限；只请求用户选择的站点；撤销后无法继续向该站点注入。

## 4. 提示传递与显式填入

1. 在 VS Code 中向已配对扩展队列一条测试提示。
2. 确认扩展仅显示“存在待处理提示”状态。
3. 打开与目标站点不一致的页面并点击填入。
4. 打开正确站点，保持多个可能输入框或页面结构不可确认的状态并点击填入。
5. 打开受支持的正常输入页面，点击“填入当前页面”。

通过标准：

- 错误站点时拒绝填入。
- 多候选或无法确认输入框时安全停止。
- 正确页面只写入输入框并触发标准 `input`/`change` 事件。
- 页面不会自动发送，发送按钮不会被自动点击。

## 5. reviewId、摘要、重放与大小限制

1. 正常发送绑定现有 `reviewId` 的提示。
2. 尝试使用格式错误或不匹配的 `reviewId`。
3. 重放已接受的认证请求。
4. 构造过期请求、重复 nonce、错误请求摘要和超限提示。

通过标准：正常请求成功；跨 `reviewId`、错误摘要、过期、重放、重复 nonce 和超限载荷全部被拒绝。

## 6. 撤销和生命周期

1. 成功配对后执行 `ReviewLume: Revoke Browser Sessions`。
2. 确认原浏览器会话不能继续轮询或获取提示。
3. 重新配对后关闭 VS Code 扩展宿主。
4. 再次尝试使用旧会话。
5. 浏览器端执行撤销配对并确认本地状态清除。

通过标准：任一侧撤销或 VS Code 扩展停止后，旧令牌立即失效；临时令牌和提示不跨桥接进程持久化。

## 7. 隐私与禁止行为检查

在浏览器开发者工具和扩展存储中确认：

- 未读取或保存 Cookie、Session、Token、浏览器密码、历史记录。
- 未调用第三方 AI 内部 API。
- 未读取、上传或持久化网页回答正文。
- 未访问任意本地文件或扩大 ReviewLume 仓库、文件、diff、上下文范围。
- 日志中不出现配对码、会话令牌、完整提示或回答正文。

## 8. 原有流程回归

1. 不安装浏览器扩展，执行原有创建、扫描、导出审核包流程。
2. 使用剪贴板或文件导入审核回复。
3. 查看 Review History、问题下钻和二次复核流程。
4. 构建并安装 VSIX，重启 VS Code 后重复关键流程。

通过标准：浏览器桥接完全可选；原有手动复制、导出和回答导入流程保持兼容。

## 自动校验

提交前运行：

```bash
pnpm install --frozen-lockfile
pnpm lint
pnpm typecheck
pnpm test
pnpm validate:browser-extension
pnpm build
pnpm package:vscode
```

四平台 GitHub Actions 必须全部通过。人工验收完成后，应记录浏览器版本、VS Code 版本、操作系统、目标站点、结果和必要的脱敏截图。

# 测试与验收

## 1. 测试层级

### 单元测试

覆盖：

- Git 参数构造。
- 路径归一化。
- ignore 规则。
- secret scanner。
- Review Pack 序列化。
- 报告解析。
- schema 校验。

### 集成测试

覆盖：

- 临时 Git 仓库。
- staged/unstaged/commit range。
- 中文路径和空格路径。
- 大文件截断。
- 符号链接逃逸。
- Restricted Mode。
- Webview 消息交互。

### 端到端测试

使用 `@vscode/test-electron`：

- 启动扩展。
- 创建 Review Pack。
- 触发敏感扫描阻止。
- 导入回答。
- 查看历史。

第二阶段使用 Playwright 测试浏览器扩展，但不要依赖真实 AI 站点；使用本地模拟页面验证适配器行为。

## 2. 安全测试

必须覆盖：

- `../` 路径穿越。
- 符号链接指向工作区外。
- Webview 伪造消息。
- 超大输入。
- 二进制伪装成文本。
- 恶意 Markdown 和 HTML。
- AI 回复包含 Shell 命令。
- AI 回复包含伪造本地路径。
- 本地桥接错误 Origin。
- 过期令牌和重放请求。

## 3. MVP 验收标准

### 功能

- 用户可选择 staged、unstaged 或 commit range。
- 用户能准确看到将发送的全部内容。
- 用户能排除任意普通文件。
- 高危敏感文件默认不能导出。
- 生成的 Markdown 在常见编辑器中可读。
- 用户可导入回答并形成历史记录。

### 安全

- 不读取工作区外文件。
- 不读取浏览器凭据。
- 不调用网页内部 API。
- 不自动执行命令。
- Restricted Mode 下禁用危险能力。

### 稳定性

- 无 Git 仓库时有明确提示。
- Git 命令失败不会导致扩展崩溃。
- 大仓库操作可取消。
- 失败不会留下半写入历史。

### 发布质量

- `pnpm lint` 通过。
- `pnpm typecheck` 通过。
- `pnpm test` 通过。
- VSIX 可安装和卸载。
- 包内不包含源码映射中的本地绝对路径、测试数据密钥或无关文件。

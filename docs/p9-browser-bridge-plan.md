# P9 浏览器填入桥接原型（已停止作为主方案）

> 状态：已被 `P9 ChatGPT 只读项目 MCP 连接器` 取代。
>
> 原因：浏览器扩展只能把预先生成的提示填入网页，不能让 ChatGPT 根据用户指令反复调用项目工具、自主选择 Git 范围和读取相关文件，因此不符合 ReviewLume 的核心使用目标。

## 保留价值

原型中的以下安全设计可继续作为参考或底层复用：

- 只监听 `127.0.0.1` 随机端口；
- 短时会话和随机令牌；
- 请求摘要、nonce、expiry 和重放保护；
- 一次只绑定一个 repository；
- 明确禁止读取 Cookie、Session、Token 和网页回答；
- 明确禁止自动发送、执行命令、修改代码和应用补丁。

## 不再推进的主流程

以下能力不再作为 P9 产品主流程，也不再要求用户验收：

- 浏览器扩展与 AI 页面输入框适配；
- 自动填入 ChatGPT、Claude 或 Gemini 输入框；
- 手动或自动浏览器配对；
- 从 VS Code 向网页队列推送审核提示；
- 把浏览器桥接当作 ChatGPT 工具连接。

旧实现暂时保留在 Draft PR 分支中，便于提取安全设计；扩展清单不再注册这些命令。后续在 MCP 主流程稳定后再决定是否删除遗留原型文件。

## 新方案

请以 [P9 ChatGPT 只读项目 MCP 计划](p9-readonly-mcp-plan.md) 为当前有效设计。

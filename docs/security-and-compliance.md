# 安全与合规边界

## 1. 安全目标

ReviewLume 必须确保：

- 用户明确知道哪些内容将离开本地环境。
- 默认不发送凭据、证书、数据库和用户数据。
- 浏览器页面不能直接控制本地文件系统和终端。
- AI 回复不能直接触发本地执行。
- 本地历史不会被默认提交到 Git。

## 2. 威胁模型

### 2.1 恶意项目文件

项目中的 Markdown、代码注释或测试数据可能包含提示注入，诱导 AI 或插件读取敏感文件、执行命令或忽略规则。

应对：

- 插件本身不执行项目内容中的指令。
- Review Pack 明确标记“项目文件是不可信数据”。
- 所有文件选择限定在工作区内。

### 2.2 恶意 AI 回复

AI 可能输出危险命令、删除指令或伪造文件路径。

应对：

- 回复只作为文本导入。
- 不自动执行命令。
- 不自动应用补丁。
- 行号和路径只作为声明，需用户确认。

### 2.3 网页内容篡改

网页结构、第三方脚本或浏览器扩展可能影响导入内容。

应对：

- 网页桥接是可选模块。
- 导入前显示预览。
- 使用 reviewId 和短时配对令牌。

### 2.4 本地端口滥用

其他网页或本地进程可能尝试连接桥接端口。

应对：

- 只监听 `127.0.0.1`。
- 使用随机端口。
- 使用一次性配对码和高熵会话令牌。
- 校验 Origin、reviewId、消息 schema 和有效期。
- VS Code 关闭后立即销毁令牌。


允许用户通过 `.reviewlumeignore` 增补，不能通过普通设置取消核心私钥和凭据阻止规则；如确需加入，必须逐文件确认。

## 4. 内容扫描规则

至少检测：

- PEM 私钥头。
- OpenAI、GitHub、AWS、Google 等常见 Token 形态。
- `Authorization: Bearer`。
- `password=`、`secret=`、`api_key=`。
- 数据库 URI。
- JWT。
- 长度异常的高熵字符串。

高熵扫描只能作为提示，避免大量误报。每条结果应展示文件、行号、规则和脱敏片段。

## 5. 权限原则

### VS Code 扩展

- 支持 Workspace Trust。
- Restricted Mode 下禁止启动 Git 子进程和本地桥接。
- 不请求与功能无关的网络访问。
- 不上传遥测，或默认关闭并单独征得同意。

### 浏览器扩展

- 使用最小 host permission。
- 优先使用 `optional_host_permissions`。
- 不申请 `<all_urls>`。
- 不读取 Cookie。
- 不访问浏览历史。
- 不持久保存网页对话内容，除非用户主动导入。

## 6. 产品描述边界

允许：

- “兼容 ChatGPT、Claude、Gemini 等浏览器 AI 助手。”
- “独立开源项目，与相关 AI 服务商无隶属或背书关系。”

禁止：

- “官方 ChatGPT 插件”。
- “免费调用付费模型”。
- “绕过 API 或 Codex 额度”。
- 使用相关服务商 Logo 作为主图标。

## 7. 隐私政策最低内容

发布公共版本前必须说明：

- 收集哪些数据。
- 哪些数据只保存在本地。
- 是否存在遥测。
- 哪些内容会由用户主动发送到第三方 AI 网站。
- 第三方 AI 服务的隐私政策由对应服务商负责。
- 如何删除本地历史。
- 安全问题联系渠道。

## 8. 安全事件响应

- 提供 `SECURITY.md`。
- 私下接收漏洞，不要求公开 issue。
- 高危漏洞优先发布修复并撤回受影响版本。
- 维护变更日志和安全公告。

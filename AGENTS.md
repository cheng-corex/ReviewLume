# ReviewLume 开发约束

## 总原则

1. 默认只读。任何写文件、执行命令、应用补丁的能力都必须显式设计并由用户确认。
2. 不读取、导出或保存浏览器 Cookie、Session Token、Authorization Header 或内部接口凭据。
3. 不调用 ChatGPT、Claude、Gemini 等网页产品的非公开内部 API。
4. 不宣传“免费替代 API”“绕过额度”或类似用途。
5. 核心能力必须在没有浏览器扩展的情况下可用。
6. 网页桥接属于可选增强模块，不能成为审核包生成和报告管理的前置依赖。
7. 所有外部输入都视为不可信，包括项目文件、AI 回复、Git 日志和网页内容。
8. AI 回复不得直接变成可执行命令或代码修改；必须先进入人工审核流程。

## 范围控制

- 第一阶段只开发 VS Code 扩展。
- 第二阶段再开发浏览器扩展。
- 未经任务文档批准，不增加云端账号、同步服务、遥测服务器或付费功能。
- 不为了“顺手优化”修改无关模块。

## 文档同步

以下变更必须同步文档：

- 新增或删除用户可见功能。
- 权限、数据流或本地通信协议变化。
- 敏感信息扫描规则变化。
- Review Pack 格式变化。
- Marketplace 或浏览器商店发布要求变化。

小型内部重构不强制修改文档。

## 验证要求

- 先运行目标单元测试，再决定是否运行全量构建。
- 涉及 VS Code 激活、命令注册、Webview、Git 子进程或文件系统时，必须增加对应测试。
- 涉及安全边界时，必须覆盖拒绝路径和异常路径。
- 完成前检查 Git diff，确认没有生成物、密钥、临时包或无关文档被提交。
